# Phase 10: Post-MVP Expansion Backlog

> Outcome: Future work is explicit, prioritized, and gated by real MVP usage.

## Rule

Do not start these until MVP demo is stable and useful.

## Candidate Expansions

1. Persistent storage (TimescaleDB/ClickHouse).
2. Multi-model and multi-tenant support.
3. UI dashboards.
4. GPU/cost analytics.
5. Python SDK ergonomics.
6. Advanced anomaly detection.

## Prioritization Framework

For each candidate, require:

- User pain evidence
- Expected impact metric
- Implementation cost estimate
- Success/kill criteria

## Exit Criteria

- Backlog is ranked by evidence, not novelty.
- At most one major expansion is active at a time.
